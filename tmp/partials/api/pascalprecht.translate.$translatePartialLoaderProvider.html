$translatePartialLoaderProvider
service in module pascalprecht.translate

Description

By using a $translatePartialLoaderProvider you can configure a list of a needed translation parts directly during the configuration phase of your application's lifetime. All parts you add by using this provider would be loaded by angular-translate at the startup as soon as possible.

Methods

  • addPart(name)

    Registers a new part of the translation table to be loaded once the angular-translate gets into runtime phase. It does not actually load any translation data, but only registers a part to be loaded in the future.

    Parameters
    ParamTypeDetails
    namestring

    A name of the part to add

    Returns
    object

    $translatePartialLoaderProvider, so this method is chainable

  • deletePart(name)

    Removes the previously added part of the translation data. So, angular-translate will not load it at the startup.

    Parameters
    ParamTypeDetails
    namestring

    A name of the part to delete

    Returns
    object

    $translatePartialLoaderProvider, so this method is chainable

  • isPartAvailable(name)

    Checks if the specific part is available. A part becomes available after it was added by the addPart method. Available parts would be loaded from the server once the angular-translate asks the loader to that.

    Parameters
    ParamTypeDetails
    namestring

    A name of the part to check

    Returns
    boolean

    Returns true if the part is available now and false if not.

  • setPart(lang, part, table)

    Sets a translation table to the specified part. This method does not make the specified part available, but only avoids loading this part from the server.

    Parameters
    ParamTypeDetails
    langstring

    A language of the given translation table

    partstring

    A name of the target part

    tableobject

    A translation table to set to the specified part

    Returns
    object

    $translatePartialLoaderProvider, so this method is chainable
