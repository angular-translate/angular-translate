自定义加载器

如果它不能提供你一个方法建立您的自定义异步装载机， angular-translate 将不再是 angular-translate。 在 异步加载 你已经学会了如何使用特定的模式使用提供的加载器从一个异步加载您的翻译资料服务，。

然而，在某些情况下，这是不够的。也许你想有一个完全不同的地方来存储您的翻译资料，甚至不希望加载它们从远程服务器。 这是自装载机进来，你可以建立你拥有装载机变得非常容易。本章将告诉您如何做到这一点！

建立一个自定义加载程序服务

如果不提供装载机适合您的需求，您可以注册一个自定义异步加载器作为一个工厂。工厂必须返回一个函数，它预期一个对象，其中至少有语言键属性存在。 与此架构你尽可能地自由并完全控制您怎样的异步加载器应该表现。

自定义加载器工厂可能看起来像这样：

app.factory('customLoader', function ($http, $q) {
    // return loaderFn
    return function (options) {
        var deferred = $q.defer();
        // do something with $http, $q and key to load localization files

        var data = {
            'TEXT': 'Fooooo'
        };

        // resolve with translation data
        return deferred.resolve(data);
        // or reject with language key
        return deferred.reject(options.key);
    };
});

你必须确保你的加载函数返回一个承诺。它应该要么获取与您的翻译数据解析，或拒绝与语言键。 如你可以看到，有个'选项'对象的'关键'属性。它具有价值语言的关键与异步加载器被调用。

创建一个自定义加载器的使用

你知道 $translateProvider 提供类似 useStaticFilesLoader()的方法或者 useUrlLoader() 的方法. 两者在内部都是用 useLoader() 去注册一个加载器共产。 这个工厂之后会被 $injector 触发。 你可以使用相同的方法来自定制服务。因此注册自定义的服务只是做：

$translateProvider.useLoader('customLoader');

angular-translate 用 $injector 服务访问自定义加载器工厂，其他的就不需要你来管了。

添加额外的选项

options.key 属性指示 options 就是一个对象。 您可以对附加属性进行扩展，以满足自己的需要。 比如你可以期待 在 options 对象上的foo, 所有您需要做的是注册您的自定义加载器工厂时提供他们的属性。

所以，如果我们需要一个属性foo 的一个值 bar 我们可以这样做：

$translateProvider.useLoader('customLoader', { foo: 'bar'});

然后，在你装载机工厂，你可以通过 options.foo 访问该属性. 您不必担心options.key 因为它在调用加载器时被 angular-translate 内部加入了。

让我们用我们的示例应用程序自定义加载程序的显示是如何工作的！首先， 我们建立这样的装载机：

app.factory('asyncLoader', function ($q, $timeout) {

  return function (options) {
    var deferred = $q.defer(),
        translations;

    if (options.key === 'en') {
      translations = {
        HEADLINE: 'What an awesome module!',
        PARAGRAPH: 'Srsly!',
        PASSED_AS_TEXT: 'Hey there! I\'m passed as text value!',
        PASSED_AS_ATTRIBUTE: 'I\'m passed as attribute value, cool ha?',
        PASSED_AS_INTERPOLATION: 'Beginners! I\'m interpolated!',
        VARIABLE_REPLACEMENT: 'Hi {{name}}',
        BUTTON_LANG_DE: 'german',
        BUTTON_LANG_EN: 'english'
      };
    } else {
      translations = {
        HEADLINE: 'Was für ein großartiges Modul!',
        PARAGRAPH: 'Ernsthaft!',
        PASSED_AS_TEXT: 'Hey! Ich wurde als text übergeben!',
        PASSED_AS_ATTRIBUTE: 'Ich wurde als Attribut übergeben, cool oder?',
        PASSED_AS_INTERPOLATION: 'Anfänger! Ich bin interpoliert!',
        VARIABLE_REPLACEMENT: 'Hi {{name}}',
        BUTTON_LANG_DE: 'deutsch',
        BUTTON_LANG_EN: 'englisch'
      };
    }

    $timeout(function () {
      deferred.resolve(translations);
    }, 2000);

    return deferred.promise;
  };
});

那么它有什么作用？它使用 $translate 超时服务，等待两秒钟，做出决议。我们无论是与英语或德语翻译表，根据承诺 对语言的关键。现在使用 $translateProvider.useLoader()，来使用：

$translateProvider.useLoader('asyncLoader');
